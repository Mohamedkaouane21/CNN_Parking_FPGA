Manuel d'utilisation
Guide de démarrage rapide
▼
Ce système détecte automatiquement les véhicules sur les places de parking grâce à une caméra IP et de l'intelligence artificielle (YOLO). Il fonctionne sur un Raspberry Pi et transmet les données via LoRa (XBee).

Étape 1 — Ajouter une caméra
Allez dans l'onglet Live View, cliquez Ajouter. Remplissez le nom, l'adresse IP de la caméra, le port (généralement 554), le chemin RTSP, les identifiants de connexion et la lettre de zone (A, B, C...). Cliquez Tester pour vérifier la connexion avant d'enregistrer.

Étape 2 — Capturer une image de référence
Allez dans l'onglet Éditeur de places. Sélectionnez votre caméra puis cliquez Capturer un screenshot. L'image apparaît dans la liste des sources.

Étape 3 — Tracer les places de parking
Sélectionnez l'image capturée. Deux modes de tracé sont disponibles :
  • Manuel : cliquez 4 coins pour chaque place, dans l'ordre (sens horaire ou anti-horaire).
  • Séparateurs : tracez des lignes entre les places (glisser-déposer). N séparateurs créent N-1 places automatiquement. Cliquez "Générer les places" pour valider.
Le mode Éditer permet de sélectionner une place, déplacer ses coins, la supprimer, ou la marquer comme gérée par un capteur externe.

Étape 4 — Sauvegarder et activer
Cliquez Sauvegarder sur le RPI pour enregistrer le masque. Puis cliquez ★ Actif pour que ce masque soit utilisé lors de la détection en direct.

Étape 5 — Configurer la caméra par défaut
Dans l'onglet Live View, sélectionnez votre caméra et cliquez ★ Caméra par défaut. Au prochain démarrage du Raspberry Pi, la détection se lance automatiquement sur cette caméra.

Étape 6 — Voir le flux en direct
Cliquez Démarrer le flux. Les places libres apparaissent en vert, les occupées en rouge. Le comptage est affiché dans la barre latérale. Quand vous arrêtez le flux, la détection continue en arrière-plan (mode headless) pour économiser les ressources.
Éditeur de places — Détail
▼
Navigation dans l'image
  • Zoom : molette de la souris ou pincement à deux doigts sur écran tactile.
  • Déplacement : clic molette + glisser (souris) ou glisser avec un doigt (tactile).
  • Plein écran : bouton en haut à droite de l'image. Sur tablette, le bouton "Quitter plein écran" apparaît en rouge en haut à droite.

Mode Manuel
Cliquez (ou tapez) sur 4 points pour former le polygone d'une place. Les points sont numérotés 1 à 4. La place est créée automatiquement après le 4e point. Clic droit (souris) ou longue pression (tactile) pour annuler.

Mode Séparateurs
Idéal pour les rangées de places alignées. Tracez des lignes de séparation en glissant du bord gauche au bord droit de la rangée. Chaque paire de lignes consécutives forme une place. Cliquez "Générer les places" quand toutes les lignes sont tracées.

Mode Éditer
Cliquez sur une place pour la sélectionner (surlignée en bleu). Glissez un coin pour le repositionner. Raccourcis :
  • Suppr : supprime la place sélectionnée
  • Capteur : marque la place comme gérée par un capteur externe (affichée en gris avec un "S"). Ces places ne sont pas comptées par la caméra.

Masques par image
Chaque image a son propre fichier de masque. Vous pouvez avoir plusieurs images de la même caméra (différents angles, résolutions) avec des masques différents. Le bouton ★ Actif définit quel masque est utilisé pour la détection en direct.

Gestion avancée (bouton ⚙)
Renommer ou supprimer des images, importer des images depuis votre ordinateur, dupliquer un masque d'une image à une autre, exporter les coordonnées en CSV.
Paramètres de détection
▼
Ancrage vertical (50-100%, défaut 85%)
Position du point de détection sur la hauteur du véhicule détecté. 50% = centre du véhicule, 100% = bas (roues). La valeur de 85% cible le bas du véhicule, ce qui est optimal pour les vues plongeantes. Ce point cyan visible sur le flux détermine si le véhicule est "dans" une place.

Ancrage horizontal (0-100%, défaut 50%)
Position horizontale du point de détection. 50% = centre. Ajustez pour les places en biais ou les caméras décalées.

Fenêtre de lissage (3-30, défaut 15)
Nombre de frames sur lesquelles le système vote pour déterminer si une place est occupée. Plus la valeur est haute, plus le résultat est stable (pas de clignotement) mais plus lent à réagir aux changements. Valeur recommandée : 10-20.

Réglages avancés (accessible via le bouton dans la section Réglages) :
  • Seuil de confiance (défaut 0.25) : confiance minimum pour qu'un véhicule soit détecté. Augmentez si trop de faux positifs.
  • Seuil NMS (défaut 0.45) : filtre les détections qui se chevauchent.
  • FPS du flux (défaut 12) : images par seconde du flux vidéo. Réduire pour économiser la bande passante.
  • Fréquence YOLO (défaut 6) : exécute la détection toutes les N frames. Plus haut = moins de charge CPU.
  • Qualité JPEG (défaut 80) : compression du flux vidéo. 30 = léger, 100 = haute qualité.
  • Seuil d'occupation (défaut 0.6) : pourcentage de votes positifs pour considérer une place occupée.
Communication XBee / LoRa
▼
Le module XBee branché en USB sur le Raspberry Pi transmet périodiquement les données de comptage via LoRa.

Configuration
Accessible via Réglages avancés → section XBee/LoRa. Activez le toggle, puis utilisez "Auto-détecter" pour trouver automatiquement le port et le baud rate du module.

Zone
Chaque caméra est associée à une lettre de zone (A-Z), configurable dans les propriétés de la caméra. Cette lettre identifie la source dans le JSON transmis.

Format des données
Toutes les 30 secondes (configurable) :
{"zone":"A","nb_places_libres":12,"nb_places_total":20,"device":"camera"}

En cas de perte du flux caméra, les valeurs passent à -1 :
{"zone":"A","nb_places_libres":-1,"nb_places_total":-1,"device":"camera"}

Le XBee démarre automatiquement au démarrage du Raspberry Pi si il était activé. Il continue d'envoyer même quand le flux vidéo est arrêté (mode headless).
Modes de fonctionnement
▼
Mode headless (arrière-plan)
Au démarrage du Raspberry Pi, la détection se lance automatiquement sur la caméra par défaut sans générer de flux vidéo. Cela économise ~30% de CPU. Les compteurs, le XBee et les alertes fonctionnent normalement. L'overlay affiche les compteurs en temps réel.

Mode streaming (flux vidéo)
Quand vous cliquez "Démarrer le flux", le système génère le flux vidéo avec l'overlay (places colorées, HUD, point d'ancrage). Plus gourmand en CPU mais permet de visualiser la détection.

Mode démo
Permet de tester la détection sur un fichier vidéo local (onglet Démo vidéo). Le lancement de la démo interrompt temporairement la détection en direct. À la fin de la démo, la détection live reprend automatiquement.

Alertes caméra
Le système surveille en continu l'état du flux :
  • Image noire : la caméra est probablement tombée, débranchée ou le capuchon est resté → alerte sonore + bannière rouge.
  • Image figée : le flux semble bloqué → bannière orange.
  • Flux perdu : reconnexion automatique en cours.
Cliquer sur la bannière la ferme. L'alerte disparaît automatiquement quand le problème est résolu.
Dépannage
▼
L'interface ne s'affiche pas
Vérifiez que le service tourne : sudo systemctl status SystemVision

YOLO non disponible
Le modèle yolov8s_visdrone_320.onnx doit être dans le dossier models/.

Flux noir ou connexion impossible
Vérifiez l'IP, le port et les identifiants de la caméra. Utilisez le bouton "Tester" dans les propriétés de la caméra. Trois niveaux d'erreur : IP injoignable, authentification échouée, pas d'image.

XBee ne démarre pas
Installez pyserial : pip install pyserial --break-system-packages. Vérifiez que le module est détecté : ls /dev/ttyUSB*

Température CPU élevée
Visible dans la barre de statut. Rouge > 75°C. Réduisez le FPS ou augmentez la fréquence YOLO dans les réglages avancés.

Masque "aucun" en live
Vous n'avez pas défini de masque actif. Allez dans l'éditeur, sauvegardez un masque et cliquez ★ Actif.

Consulter les logs
journalctl -u SystemVision -f
Filtrer par composant : | grep YOLO, | grep XBEE, | grep LIVE
